# Outputs Folder

This folder is used by the pipeline to store runtime evidence files generated during local execution.

At submission time, this folder contains only this README file. When the project is reproduced with Docker Compose, the scripts automatically create output files in this directory.

## Generated Runtime Files

The following files may be created during a pipeline run:

| File | Created by | Purpose |
|---|---|---|
| `load_summary.json` | `scripts/load_data.py` | Summarizes the batch loading run |
| `load_checkpoint.json` | `scripts/load_data.py` | Stores the last successfully completed batch |
| `data_quality_report.json` | `scripts/load_data.py` | Summarizes rejected rows, invalid values, and quality flags |
| `invalid_rows.jsonl` | `scripts/load_data.py` | Stores rejected rows and rejection reasons |
| `load_failures.json` | `scripts/load_data.py` | Records failed batch write attempts if a failure occurs |
| `validation_summary.json` | `scripts/validate_data.py` | Summarizes database validation results |
| `example_query_results.json` | `scripts/example_queries.py` | Stores the results of example monitoring queries |

## Role in the Project

The output files provide evidence that the system was executed successfully. In particular:

- `load_checkpoint.json` demonstrates checkpoint-based batch recovery.
- `data_quality_report.json` documents data-quality consequences.
- `validation_summary.json` confirms that the MongoDB collection was checked after loading.
- `example_query_results.json` demonstrates how the stored data can support monitoring and maintenance use cases.

## Reproduction

The main reproduction workflow is described in the root `README.md`. The most important commands are:

```bash
docker compose run --rm app python scripts/load_data.py --reset-checkpoint --max-batches 2
docker compose run --rm app python scripts/load_data.py --max-batches 2
docker compose run --rm app python scripts/validate_data.py
docker compose run --rm app python scripts/example_queries.py
```

After these commands run successfully, this folder will contain the generated runtime evidence files.

## Submission Note

Generated output files are runtime artifacts. They are useful for local evidence and screenshots, but they do not need to be stored in the clean submitted repository because they can be regenerated by running the reproduction commands.

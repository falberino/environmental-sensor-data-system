# Phase 3 Run Evidence

Demonstration plan for the **environmental-sensor-data-system** portfolio submission.

This workflow proves the loader processes a **small part** of the CSV, **stops intentionally**, **saves a checkpoint**, and **continues later** without reprocessing earlier batches.

---

## Recommended demonstration (partial batch + resume)

Run from the project root after `cp .env.example .env`.

```bash
docker compose down -v
docker compose up -d mongodb
docker compose run --rm app python scripts/init_db.py
docker compose run --rm app python scripts/load_data.py --reset-checkpoint --max-batches 2
docker compose run --rm app python scripts/load_data.py --max-batches 2
docker compose run --rm app python scripts/validate_data.py
docker compose run --rm app python scripts/example_queries.py
pytest tests/ -v
```

### What each step demonstrates

| Step | Evidence |
|------|----------|
| `docker compose down -v` | Clean MongoDB volume for reproducible demo |
| `init_db.py` | Indexes on `city_environment.sensor_measurements` |
| `load_data.py --reset-checkpoint --max-batches 2` | First **2 batches** (~2000 rows); checkpoint at `outputs/load_checkpoint.json` |
| `load_data.py --max-batches 2` | **Resume**: skips batches 1–2, processes batches 3–4 |
| `validate_data.py` | Quality field, null counts, device density, `ready_for_queries` |
| `example_queries.py` | Downstream alert and aggregation queries |
| `pytest tests/ -v` | Transformation and checkpoint unit tests |

### Files to capture as evidence (after the run)

| File | Purpose |
|------|---------|
| `outputs/load_checkpoint.json` | `last_successful_batch`, `rows_processed` |
| `outputs/load_summary.json` | Per-run load statistics |
| `outputs/data_quality_report.json` | Rejected rows, quality flags, consequences |
| `outputs/validation_summary.json` | Post-load validation |
| `outputs/example_query_results.json` | Example query output |

These files are gitignored and regenerated by the commands above.
